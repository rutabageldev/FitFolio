### Test review checklist

- Purpose clear and behavior-focused
- Correct level and bucket; descriptive names
- Proper markers (`unit`, `integration`, `security`, etc.)
- Minimal mocking; no-mock path exists for critical flows
- Deterministic (time/random/env controlled)
- Uses shared fixtures; domain fixtures documented
- Parametrization used where appropriate; no duplication
- Coverage impact acceptable; no decrease in touched modules
- Docs updated if public behavior changed
